# Reflection & Learnings

## 1. The Hardest Bug I Faced This Week

One of the most difficult issues I faced was during the deployment phase on Vercel. The application worked perfectly in the local environment, but after deploying to production, the AI summary generation and email-sending functionality stopped working.

Initially, I assumed the issue was related to API keys or environment variables. After debugging multiple times, I researched how Vercel handles serverless functions and discovered that imported files inside serverless functions needed to resolve correctly after build output generation. I found that importing `.ts` files directly inside the serverless function caused issues in production, so I updated the imports to `.js`.

Even after fixing that, the problem persisted. After several trial-and-error attempts, I discovered that I had created an additional `lib` folder inside the root `api` folder, which caused Vercel’s serverless function resolution to fail. Moving the folder outside the `api` directory fixed the issue completely.

This debugging process taught me how Vercel structures and executes serverless functions internally and improved my understanding of deployment environments versus local development environments.

## 2. A Decision I Reversed Mid-Week

Midway through development, I decided to redesign the audit recommendation logic completely.

Initially, I generated most of the logic with the help of AI tools. However, after testing the recommendations against real use cases, I realized the outputs were not properly aligned with the project goals and contained several logical inconsistencies.

To improve reliability, I manually restructured the audit flow and rewrote the recommendation logic based on actual usage patterns, subscription comparisons, and realistic cost-saving scenarios.
